# LexVision POC

## Detailed working project

Working detail for this initiative lives in the Claude Code project at `~/claude_code/projects/lexvision-poc` (the `project_repo` field above). That project owns the working detail; this note holds the hub layer. Per-topic detail:

- `docs/SOURCES.md` - provenance and up-link map back to this note.
- `docs/architecture.md` - system architecture, components, data flow (patent figures mapped to implementation).
- `docs/ontology.md` - ontological model: entity, relationship, role, right and obligation types.
- `docs/data-model.md` - graph schema (node labels, relationship types, properties, Cypher examples).
- `docs/poc-scope.md` - what is in and out of scope, success criteria, known simplifications.
- `docs/decisions.md` - architecture decision records (append-only).
- `docs/benchmarking.md` - per-component evaluation approach and benchmark register.
- `docs/glossary.md` - patent terminology mapped to implementation concepts.
- `docs/kumulus/` - the Kumulus (Logicalis) vendor dossier: dated meeting notes, proposal assessments, and research.
- `docs/IDEAS.md` - project backlog.
