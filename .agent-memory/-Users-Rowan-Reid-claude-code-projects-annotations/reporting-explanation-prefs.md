When presenting MRI-133 PoC results, show every percentage with its complementary share AND name that complement: e.g. queue 22% has an "auto-answered" 78% remainder; precision 93% has a "confidently-wrong" 7% remainder; abstain-ok 81% is a slice inside the queue, not its size. Never leave a remainder implicit. Anchor each ratio to its own denominator (all pages / answered / actionable / queued) so both sides are on screen.

Prefer plain-English, layman framing over jargon for this project, with a one-line "how it works" per matcher and a scenario glossary in reports. Rowan is not a Python developer, so explain via analogies, not code. He often consumes deep dives as audio (co-pilot audio mode).

**Why:** emerged strongly 2026-07-01 while building the visual bake-off HTML report (`poc/data/bakeoff.html`, generated by `scoring/report_html.py`); he explicitly asked for complement-visible percentages and a glossary.

**How to apply:** in any report or metric view for this project, pair each rate with its named complement, keep the master page-split (confident-correct / silent-error / safe-abstain / over-abstain) visible, and add plain definitions. Candidate for promotion to the profile plain-explanation register if it recurs cross-project. Links: [[mri-133-project]].
